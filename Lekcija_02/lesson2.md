## 2. Регламент (ранее «Тренажёр»)

### Что это
**Регламент** — рабочий документ, созданный на основе транскрибации лекции. Содержит знания, инструменты и правила, на которые можно ссылаться при разработке.

### Как создать
1. Транскрибировать запись лекции
2. Передать транскрипцию в Antigravity с промптом:  
   *«Мы прошли лекцию по веб-кодингу. Сделай полноценный рабочий документ, на который я буду ссылаться при разработке продуктов»*
3. Сохранить результат как файл

### Как использовать
- **В новом проекте:** Подключить файл регламента → написать *«Используй этот регламент для разработки»* → агент будет следовать правилам
- **Объединение (мердж):** После каждой лекции создавать новый регламент, затем объединять с предыдущим: *«Объедини эти два регламента»*
- **Customization (Setting):** В настройках Antigravity → Setting → Customization → Manage — можно вставить правила на уровне программы (пока рано, мало материала)

### Ключевые принципы
- Регламент **растёт** от лекции к лекции (мердж = объединение данных)
- Можно **дополнять** своими открытиями (например: «Не упрощай мою идею, а структурируй»)
- К концу обучения — полноценный пакет правил для разработки

## 3. Skills (Скиллы / Субагенты)

### Что это
**Скиллы** — вспомогательные инструменты для Antigravity. Это не простые промпты, а полноценные пакеты с внутренней кодовой базой, предназначенные для конкретных задач: копирайтинг, дизайн, фронтенд, маркетинг и т.д.

- На момент лекции: **84 000+** скиллов
- Сайт: **skills.sh** (требуется VPN)
- Иногда называют «субагенты»

### Принцип работы
Основной агент (Antigravity) может **делегировать** задачи субагентам (скиллам). Субагент видит файловую директорию проекта, понимает контекст и даёт более точные результаты.

### Установка скиллов

**Через терминал:**
1. Зайти на skills.sh → выбрать нужный скилл
2. Скопировать команду установки (наверху страницы скилла)
3. Вставить в терминал Antigravity → выполнить

**Два режима установки:**

| Режим | Где живёт | Когда использовать |
|---|---|---|
| **В проект** (локально) | Папка `.agents` в директории проекта | Специфичные скиллы под конкретный проект |
| **Глобально** | На уровне ОС / Antigravity | Универсальные скиллы (FindSkill, ревью кода и т.д.) |

> ⚠️ **Осторожно с глобальной установкой** — скилл будет влиять на все проекты. Убедитесь, что он универсальный.

### Удаление скиллов
- **Локальные:** Удалить папку скилла из файловой директории
- **Глобальные:** Команда в терминале: `npm rm <название_скилла>` (или аналогичная)
- Можно спросить Antigravity: *«Какие скиллы установлены глобально?»*

### Ключевой скилл: FindSkills
Специальный скилл-поисковик (400 000+ скачиваний). Устанавливается **глобально**. Позволяет писать запросы типа: *«Найди мне нужные скиллы для создания сайта»* — и он сам найдёт и предложит.

### Лайфхак от Алексея
1. Установить FindSkills в проект
2. Попросить: *«Изучи мой проект и предложи, какие скиллы помогут»*
3. Он создаст файл с рекомендациями (9 скиллов)
4. Скинуть файл в диалог: *«Установи все эти скиллы локально в проект»*
5. Один раз Accept → всё установлено

### Скиллы vs Плагины
- **Плагины** — программные расширения Antigravity с визуализацией (≈50 шт.)
- **Скиллы** — текстовые инструкции в формате Markdown, на которые агент ссылается как на промпты

## 4. MCP (Model Context Protocol)

### Что это
**MCP** — протокол, созданный Anthropic (создатели Claude). Это «умный аналог API», который позволяет агенту напрямую подключаться к внешним сервисам без написания кода.

### API vs MCP

| | API (старый подход) | MCP (новый подход) |
|---|---|---|
| Подключение | Отдельный код для каждой сущности | Единое подключение ко всей системе |
| Формат | Код, скрипты, документация | JSON, автоматическая маршрутизация |
| Настройка | Ручная, много строк кода | Минимальная, агент разбирается сам |
| Ошибки | Часты при ручной настройке | Практически нет |

### Пример: отчёт из Битрикс24
- **API:** Подключаться отдельно к деньгам, задачам, сделкам, увольнениям → писать код для каждого → скрипт → таблица
- **MCP:** Агент → запрос → MCP-сервер сам находит нужное → JSON → таблица. Одним запросом.

### Как подключить MCP-серверы

1. **Через Antigravity:** Написать *«Помоги мне установить все необходимые MCP-серверы для продвинутой разработки»*
2. **Через Context7** (context7.com): Скопировать ссылку → скормить Antigravity → он сам установит
3. **Вручную:** Settings (справа внизу) → MCP Tools

### Популярные MCP-серверы

| Сервер | Назначение |
|---|---|
| **Supabase** | База данных — создание/миграция таблиц без ручного SQL |
| **Figma** | Дизайн — прямое подключение к макетам |
| **Next.js** | Фронтенд-фреймворк |
| **Context7** | Агрегатор библиотек и документации |
| **PostgreSQL** | База данных |

### Важно
- MCP используется **только при разработке** — в готовом продукте всё работает по обычному API
- Сервис должен **поддерживать MCP** (не все пока поддерживают, например, Битрикс24 — нет)
- Агент **сам выбирает**, какой MCP-сервер использовать для конкретной задачи

## 5. Параллельная работа в Antigravity

В Antigravity можно открыть **второго агента** (Open Manager → правая верхняя часть экрана) и работать параллельно:
- Основной агент собирает приложение
- Второй агент пишет документацию / маркетинговую стратегию / другой проект

> ⚠️ Не давайте двум агентам похожие задачи в одном проекте — возможен конфликт (будут исправлять друг друга).

## 6. Дополнительные заметки

- **Идею нужно прописывать детально самому** — AI упрощает, а не раскрывает. Совет: *«Структурируй мою идею, а не изменяй и не упрощай»*
- **Дебаггинг** — нормальная часть работы. Не переделывать проект с нуля, а исправлять конкретные баги
- **Скиллы для ревью** — можно установить скиллы, которые проверяют код и делают аудит
- **GitHub** — на 4-5 занятии будем заливать код в репозиторий. Репозиторий = хранилище кода
- **Автоподтверждение команд** — полезный плагин для автоматического accept команд в терминале
