## 🔄 Создание и использование Регламента

### Что такое Регламент
Регламент — это рабочий документ, на который ты ссылаешься при разработке. Он содержит правила, инструменты и знания, накопленные за время обучения. Растёт от лекции к лекции.

### Как создать Регламент из лекции
1. Транскрибируй запись лекции (Whisper-бот)
2. Передай транскрипцию в Antigravity
3. Напиши промпт:

```
Мы прошли лекцию по веб-кодингу. В ней преподаватель описал 
важные знания и инструменты для создания цифровых продуктов.
Сделай полноценный рабочий документ, на который я буду ссылаться 
при разработке продуктов.
```

4. Сохрани результат как файл

### Как использовать Регламент

**Способ 1 — Подключение к проекту:**
1. Открой новый чат в Antigravity
2. Прикрепи файл регламента
3. Напиши: *«Используй этот регламент для разработки. Хочу сделать [описание продукта]»*
4. Агент будет следовать правилам из регламента

**Способ 2 — Встроить в Antigravity (продвинутый):**
1. Settings (правый нижний угол) → Customization → Manage
2. Вставить правила регламента → они будут работать во всех проектах
3. ⚠️ Используй, когда регламент хорошо проработан (не сейчас)

### Мердж регламентов
После каждой новой лекции:
1. Создай новый тренажёр/регламент
2. Возьми предыдущий регламент + новый
3. Закинь оба в Antigravity:

```
Объедини эти два регламента в один. Назови "Регламент".
Сохрани все полезные знания, правила и инструменты из обоих.
```

### Чеклист:

- [ ] Объединён с предыдущим регламентом
- [ ] Файл сохранён и доступен для использования

## 🧩 Работа со Skills (Скиллами)

### Что такое скиллы
Скиллы (субагенты) — вспомогательные инструменты для Antigravity. Каждый скилл — полноценный пакет с внутренней кодовой базой, предназначенный для конкретной задачи.

- **Сайт:** [skills.sh](https://skills.sh) (нужен VPN)
- **Количество:** 84 000+ скиллов
- **Категории:** копирайтинг, дизайн, фронтенд, маркетинг, SEO, мозговой штурм и т.д.

### Как найти нужный скилл

**Ручной поиск:**
1. Зайди на skills.sh
2. Используй поиск или раздел Trending
3. Изучи описание и количество скачиваний (больше = надёжнее)

**Автоматический поиск (рекомендуется):**
1. Установи скилл **FindSkills** глобально
2. Напиши Antigravity: *«Найди мне нужные скиллы для [описание задачи]»*
3. Он подберёт и предложит установку

### Как установить скилл

**Шаг 1:** Скопируй команду установки со страницы скилла на skills.sh

**Шаг 2:** Открой терминал в Antigravity (Terminal → New Terminal)

**Шаг 3:** Вставь команду и выполни

**Шаг 4:** Выбери режим установки:

| Режим | Описание | Когда использовать |
|---|---|---|
| **Project** (локально) | Скилл появится в папке проекта | Специфичные скиллы под конкретный проект |
| **Global** (глобально) | Скилл встраивается в Antigravity | Универсальные скиллы (FindSkills, ревью кода) |

### Как использовать установленный скилл

**Локальный скилл:** Перетащи папку скилла в чат или сошлись на неё:
```
Используя скилл [название скилла], сделай мне [задачу]
```

**Глобальный скилл:** Работает автоматически в фоне. Агент сам решает, когда использовать.

### Как удалить скилл

- **Локальный:** Удалить папку из файловой директории проекта
- **Глобальный:** Команда в терминале:
```
npx ... rm <название_скилла>
```
- Или спроси Antigravity: *«Какие скиллы установлены глобально? Удали [название]»*

### Лайфхак: массовая установка скиллов
1. Установи FindSkills в проект
2. Напиши: *«Изучи мой проект и предложи, какие скиллы помогут»*
3. Он создаст файл с рекомендациями
4. Скинь файл в диалог: *«Установи все эти скиллы локально в этот проект»*
5. Один раз Accept → всё установлено

### Скиллы vs Плагины

| | Скиллы | Плагины |
|---|---|---|
| Количество | 84 000+ | ≈50 |
| Формат | Markdown-файлы (инструкции) | Программные расширения |
| Влияние | Поведение агента при разработке | Оболочка и интерфейс Antigravity |
| Установка | Терминал / skills.sh | Магазин расширений Antigravity |

## 🔌 Работа с MCP (Model Context Protocol)

### Что такое MCP
MCP — протокол, созданный Anthropic. Позволяет агенту подключаться к внешним сервисам напрямую, без написания кода для каждой интеграции.

### Разница: API vs MCP

```
API (старый подход):
  Агент → код подключения к сервису А
         → код подключения к сервису Б
         → код подключения к сервису В
         → ... (для каждого отдельный скрипт)

MCP (новый подход):
  Агент → MCP-сервер → автоматически подключается ко всему внутри сервиса
```

### Зачем нужен MCP при разработке

**Пример без MCP (база данных Supabase):**
1. Зайти в Supabase
2. Написать SQL-запрос на создание таблицы
3. Скопировать код миграции из Antigravity
4. Вставить в SQL Editor в Supabase
5. Запустить
6. Повторять при каждом изменении структуры

**Пример с MCP:**
1. Один раз подключить MCP-сервер Supabase
2. Antigravity сам создаёт/изменяет таблицы, когда нужно
3. Вы ничего дополнительно не делаете

### Как подключить MCP-серверы

**Способ 1 — Автоматический:**
```
Я хочу продвинуто разрабатывать в Antigravity.
Помоги мне установить все необходимые MCP-серверы.
```

**Способ 2 — Через Context7:**
1. Зайди на [context7.com](https://context7.com)
2. Скопируй ссылку
3. Скорми Antigravity: *«Подключи MCP-серверы отсюда [ссылка]»*
4. Он попросит авторизоваться / дать ключ → следуй инструкциям

**Способ 3 — Вручную:**
- Settings → MCP Tools → добавить вручную

### Основные MCP-серверы

| Сервер | Для чего |
|---|---|
| **Supabase** | Работа с базой данных без ручного SQL |
| **Figma** | Подключение к дизайн-макетам |
| **Next.js** | Фронтенд-фреймворк для сайтов |
| **Context7** | Агрегатор документации и библиотек |
| **PostgreSQL** | Работа с базой данных |

### Важные правила
- MCP работает **только при разработке** — в готовом продукте используется обычный API
- Сервис должен **поддерживать MCP** (не все пока поддерживают)
- Агент **сам выбирает**, какой MCP использовать (когда несколько подключены)
- Можно установить **все доступные** — лишние просто не будут использоваться

- [ ] Проверена работа: Settings → MCP Tools

## 🔀 Параллельная работа в Antigravity

### Второй агент
Можно открыть второго агента для параллельной работы:
1. Подведи мышку к **правому верхнему углу** экрана
2. Нажми **Open Manager**
3. Откроется дополнительное окно чата

### Правила параллельной работы
- ✅ Разные задачи в одном проекте (код + документация)
- ✅ Работа с разными проектами одновременно
- ❌ Похожие задачи в одном проекте (фронтенд + бэкенд) — будут конфликтовать

## 🧠 Полезные советы из лекции

- **Регламент** — ваш главный помощник, обновляйте его после каждой лекции
- **Идею описывайте сами и детально** — просите AI структурировать, а не упрощать
- **Дебаг — это нормально** — не переделывайте проект, исправляйте конкретные баги
- **Если ступор** — скриншот экрана → Antigravity → *«Что мне дальше делать?»*
- **Файлы на латинице** (Windows) — избежите проблем с путями
- **Не закрывайте локальный сервер** — тогда не придётся перезапускать проект
- **Скиллы для ревью** — установите скиллы проверки кода, они ловят ошибки
- **Новое окно (File → New Window)** — открывает отдельный экземпляр Antigravity
- **Терминал** — привязан к конкретному проекту (папке)
