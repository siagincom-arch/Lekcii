# 📖 Словарь терминов — Vibecoding (Лекции 1–4)

> Все термины объяснены простым языком для тех, кто не знает английского и не имеет опыта в программировании.

---

## 🔤 Основные термины

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Vibecoding** (Вайбкодинг) | вайб-ко́динг | англ. vibe (атмосфера) + coding (написание кода) | Метод создания программ и приложений, где человек описывает свою идею словами, а искусственный интеллект сам пишет код. Как если бы вы объясняли строителю, что хотите, а он сам строит. |
| **AI / ИИ** (Эй-Ай) | эй-а́й | англ. Artificial Intelligence | Искусственный интеллект — компьютерная программа, которая умеет «думать»: анализировать текст, писать код, отвечать на вопросы. Например, ChatGPT или Claude. |
| **Агент** (Agent) | а́гент | англ. agent (действующее лицо) | Программа-помощник на основе ИИ, которая сама выполняет задачи: ищет информацию, пишет код, запускает команды. Вы даёте ей задание — она выполняет. |
| **Субагент** (Subagent) | суб-а́гент | англ. sub (под) + agent | Вспомогательный агент. Основной агент может поручить часть работы субагенту. Как начальник делегирует задачу помощнику. В Antigravity субагенты = скиллы. |
| **Промпт** (Prompt) | промпт | англ. prompt (подсказка, запрос) | Текстовый запрос, который вы пишете искусственному интеллекту. Например: «Сделай мне сайт для кафе» — это промпт. Чем точнее промпт, тем лучше результат. |

## 🛠 Инструменты и платформы

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Antigravity** (Антигравити) | антигра́вити | англ. antigravity (антигравитация) | Программа для разработки (IDE — среда разработки). Похожа на текстовый редактор, но с встроенным ИИ-помощником. Вы пишете ему задания — он пишет код. Работает на базе VS Code. |
| **IDE** (Ай-Ди-И) | ай-ди-и́ | англ. Integrated Development Environment | «Рабочий стол программиста» — программа, в которой пишут код. Как Word для текстов, только для программирования. Antigravity — это IDE. |
| **VS Code** (Ви-Эс Код) | ви-эс ко́д | Visual Studio Code | Популярная бесплатная программа для написания кода от Microsoft. Antigravity построен на её основе. |
| **Claude / Claude Opus** | клод / клод о́пус | название от компании Anthropic | Искусственный интеллект (языковая модель), который умеет писать код, анализировать данные и вести диалог. Основной «мозг» в Antigravity. |
| **Gemini** (Джемини) | дже́мини | англ. gemini (близнецы) | ИИ от Google. Используется как запасной вариант, когда Claude зависает. Хорошо работает с большими файлами. |
| **LLM** (Эл-Эл-Эм) | эл-эл-э́м | англ. Large Language Model | Большая языковая модель — тип ИИ, который обучен на огромном количестве текстов и умеет генерировать текст и код. Claude, Gemini, ChatGPT — это всё LLM. |
| **OpenRouter** (ОпенРоутер) | о́пен-ро́утер | англ. open (открытый) + router (маршрутизатор) | Сервис-посредник, через который можно подключаться к разным ИИ-моделям через один вход. Как универсальный пульт для разных телевизоров. |
| **MiniMax** (МиниМакс) | мини-ма́кс | англ. mini + max | Модель ИИ для автоматических задач. Дешевле и быстрее Claude, но выполняет работу на схожем уровне. |
| **Supabase** (Супабейс) | супа-бе́йс | от англ. super + base (база) | Онлайн-сервис для хранения данных (база данных). Как таблица Excel, только мощнее и доступна из интернета. Заменяет Google Sheets для серьёзных проектов. |
| **GitHub** (ГитХаб) | гит-ха́б | англ. git (система контроля) + hub (центр) | Сайт-хранилище для кода. Программисты загружают туда свои проекты, чтобы хранить историю изменений и делиться кодом с другими. Как облачный диск, но специально для кода. |
| **Git** (Гит) | гит | от имени создателя Линус Торвальдс | Система контроля версий — программа, которая запоминает все изменения в файлах. Можно вернуться к любой предыдущей версии, как в машине времени для кода. |
| **Whisper-bot** (Виспер-бот) | ви́спер-бот | англ. whisper (шёпот) + bot (робот) | Программа, которая превращает аудио и видео в текст. Наговорили на диктофон — Whisper превратит в текстовый файл. |
| **n8n** (Эн-Эйт-Эн) | эн-эйт-эн | сокращение от «nodemation» | Программа для автоматизации рабочих процессов. Позволяет связать разные сервисы между собой без написания кода. Как конструктор: соединяете блоки — и они работают вместе. |
| **Perplexity** (Перплексити) | пёрплексити | англ. perplexity (недоумение) | Поисковик на основе ИИ. Ищет информацию в интернете и даёт готовые ответы с источниками. Используется для поиска проектов на GitHub. |
| **Context7** (Контекст7) | ко́нтекст-семь | англ. context (контекст) + 7 | Сервис-агрегатор документации и библиотек. Помогает ИИ быстро находить нужную информацию о программных инструментах. |
| **Figma** (Фигма) | фи́гма | название компании | Онлайн-программа для создания дизайна (макетов) сайтов и приложений. Как Photoshop, но в браузере и заточен под дизайн интерфейсов. |
| **OpenClaw / Clobot** | о́пен-кло / кло́бот | англ. claw (коготь) | Автономный агент-валидатор: следит за работой основного ИИ, проверяет результаты и создаёт отчёты. Как контролёр качества. Бесплатная, но менее безопасная альтернатива Claude Code. |
| **Claude Code** (Код-Код) | клод код | название инструмента | Терминальная программа-агент от Anthropic. В отличие от Antigravity, работает прямо в консоли, не имеет кнопок отмены и может запускать целую команду агентов параллельно. |
| **Skill Creator** | скилл криэ́йтор | англ. skill (навык) + creator (создатель) | Инструмент для Claude Code, который сам создаёт, тестирует и выбирает лучший скилл (навык) для вашей задачи. Внутренний A/B-тестировщик скиллов. |
| **Gladia** (Гладия) | гла́дия | название сервиса | Сервис для голосового ввода, встроенный в разработку. Нажимаете кнопку, говорите голосом — он переводит это в безупречный текст для ИИ-агентов. |
| **Groq / Whisper** | грок / ви́спер | названия технологий | Системы распознавания речи (превращают голос в текст). Часто используются для удобства общения с нейросетями без набора на клавиатуре. |

## 🌐 Веб-разработка

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Фронтенд** (Frontend) | фронт-э́нд | англ. front (передняя часть) + end (конец) | Всё, что видит пользователь на экране: кнопки, тексты, картинки, дизайн сайта. «Лицо» приложения. |
| **Бэкенд** (Backend) | бэк-э́нд | англ. back (задняя часть) + end (конец) | Невидимая часть приложения: серверы, базы данных, логика работы. «Мозг» приложения, который хранит данные и обрабатывает запросы. |
| **Лендинг** (Landing page) | лэ́ндинг | англ. landing (посадочная) | Одностраничный сайт, который создаётся для конкретной цели: продать товар, собрать заявки, рассказать о продукте. Вы «приземляетесь» на эту страницу из рекламы. |
| **UI** (Ю-Ай) | ю-а́й | англ. User Interface (интерфейс пользователя) | Внешний вид приложения: кнопки, меню, поля ввода — всё, с чем пользователь взаимодействует. |
| **UX** (Ю-Экс) | ю-э́кс | англ. User Experience (опыт пользователя) | Удобство использования приложения. Насколько легко и приятно пользователю выполнять свои задачи. |
| **URL** (Ю-Ар-Эл) | ю-ар-э́л | англ. Uniform Resource Locator | Адрес в интернете. Например: `https://google.com` — это URL. Каждая страница в интернете имеет свой уникальный URL. |
| **Next.js** (Некст-Джей-Эс) | некст-джей-э́с | англ. next (следующий) + JS (JavaScript) | Популярный инструмент (фреймворк) для создания сайтов. Помогает делать быстрые и красивые веб-приложения. |
| **Фреймворк** (Framework) | фре́ймворк | англ. framework (каркас) | Набор готовых инструментов и правил для создания программ. Как конструктор LEGO — вместо того чтобы делать каждый кирпичик с нуля, берёте готовые детали. |
| **Хостинг** (Hosting) | хо́стинг | англ. hosting (размещение) | Сервис, который хранит ваш сайт/приложение и делает его доступным в интернете 24/7. Как аренда квартиры для вашего сайта. |
| **Vercel** (Версе́ль) | версе́ль | название компании | Популярный сервис хостинга для сайтов. Загружаете код — и сайт сразу доступен в интернете. |
| **Railway** (Рэйлвей) | рэ́йлвей | англ. railway (железная дорога) | Сервис для запуска приложений в облаке. Альтернатива Vercel для более сложных проектов. |
| **VPS** (Ви-Пи-Эс) | ви-пи-э́с | англ. Virtual Private Server | Виртуальный приватный сервер — удалённый компьютер, который вы арендуете для размещения своих проектов. Как свой маленький компьютер в дата-центре. |

## 💻 Программирование (общие термины)

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Стек** (Stack) | стек | англ. stack (стопка) | Набор технологий и инструментов, которые используются в проекте. «Какой у тебя стек?» = «Какие программы и языки ты используешь?» |
| **Код** (Code) | код | англ. code (код, шифр) | Текст, написанный на языке программирования, который компьютер понимает и выполняет. Это инструкции для машины. |
| **Кодогенерация** | кодогенера́ция | код + генерация | Автоматическое создание кода с помощью ИИ. Вместо того чтобы писать код руками, ИИ генерирует его по вашему описанию. |
| **Скрипт** (Script) | скрипт | англ. script (сценарий) | Небольшая программа, которая выполняет конкретную задачу. Как рецепт — набор шагов, которые компьютер выполняет по порядку. |
| **Баг** (Bug) | баг | англ. bug (жучок) | Ошибка в программе. Когда что-то работает не так, как должно — это баг. Название пошло от настоящего жучка, который попал в компьютер и вызвал сбой. |
| **Дебаг / Дебаггинг** (Debug) | деба́г / деба́ггинг | англ. debug (убрать жучка) | Процесс поиска и исправления ошибок (багов) в программе. Как ремонт: нашёл поломку — починил. |
| **Фикс** (Fix) | фикс | англ. fix (исправить) | Исправление бага. «Зафиксить баг» = «исправить ошибку». |
| **Итерация** (Iteration) | итера́ция | лат. iteratio (повторение) | Один цикл повторяющегося процесса. В разработке: сделали продукт → протестировали → улучшили → это одна итерация. Потом ещё одна, и ещё. Каждый раз продукт становится лучше. |
| **Деплой** (Deploy) | депло́й | англ. deploy (развёртывать) | Размещение готовой программы на сервере, чтобы ею могли пользоваться другие люди через интернет. Как открытие магазина для покупателей. |
| **Продакшн** (Production) | прода́кшн | англ. production (производство) | Рабочая среда, где приложение доступно пользователям. «Вывести на продакшн» = «запустить для реальных пользователей». |
| **Релиз** (Release) | рели́з | англ. release (выпуск) | Официальный выпуск программы или её новой версии для пользователей. |
| **Прототип** (Prototype) | прототи́п | греч. prototypon (первообраз) | Первая, пробная версия продукта. Не финальная, но позволяет проверить идею и показать заказчику. |
| **API** (Эй-Пи-Ай) | эй-пи-а́й | англ. Application Programming Interface | Способ, которым программы «разговаривают» друг с другом. Как официант в ресторане: вы говорите заказ ему, он передаёт на кухню и приносит результат. |
| **JSON** (Джейсон) | дже́йсон | англ. JavaScript Object Notation | Формат хранения данных, понятный и людям, и компьютерам. Как заполненная анкета с полями: имя, возраст, адрес — только для программ. |
| **SQL** (Эс-Кью-Эл) | эс-кью-э́л | англ. Structured Query Language | Язык для работы с базами данных. С его помощью можно добавлять, искать и изменять данные в таблицах. Как умный поиск по Excel-таблице. |
| **Терминал** (Terminal) | термина́л | англ. terminal (конечная точка) | Текстовое окно, в которое вводят команды для компьютера. Вместо нажатия кнопок мышью — пишете команды текстом. Как командная строка Windows, но более мощная. |
| **npm** (Эн-Пи-Эм) | эн-пи-э́м | англ. Node Package Manager | Программа для установки дополнительных инструментов (пакетов) в проект. Как магазин приложений, но для программистов. Команда `npm run` запускает проект. |
| **Токен** (Token) | то́кен | англ. token (жетон, ключ) | Секретный ключ для доступа к сервису. Как пароль, но для программ. Например, токен GitHub позволяет вашей программе загружать код на сайт. |
| **Переменные окружения** (.env) | — | англ. environment variables | Секретные настройки проекта (пароли, ключи), которые хранятся в специальном файле `.env`. Этот файл никогда не загружается в GitHub, чтобы секреты не утекли. |
| **Эндпоинт** (Endpoint) | э́ндпоинт | англ. end (конец) + point (точка) | Конкретный адрес, через который программа получает или отправляет данные. Как номер телефона: звоните на определённый номер — получаете определённую информацию. |
| **Edge-кейс** (Edge case) | эдж-ке́йс | англ. edge (край) + case (случай) | Необычная, крайняя ситуация, которую программа должна уметь обработать. Например: «А что если пользователь введёт пустое поле?» |

## 🔧 Git и GitHub (команды)

| Термин | Произношение | Простое объяснение |
|--------|-------------|-------------------|
| **Репозиторий** (Repository / Repo) | репозито́рий | Хранилище проекта на GitHub. Содержит все файлы и всю историю изменений. Как папка в облаке с функцией «машины времени». |
| **git clone** (гит клон) | гит клон | Команда для скачивания проекта с GitHub на свой компьютер. Создаёт полную копию репозитория. |
| **git add** (гит эд) | гит эд | Команда, которая говорит Git: «Запомни эти файлы для следующего сохранения». Как выделение файлов перед копированием. |
| **git commit** (гит коммит) | гит ко́ммит | Команда для «фотографирования» текущего состояния проекта. Коммит = одно сохранение с описанием, что изменилось. |
| **Коммит** (Commit) | ко́ммит | англ. commit (зафиксировать) — сохранение изменений в проекте с пометкой, что было сделано. Как «Сохранить» в Word, но с подписью. |
| **git push** (гит пуш) | гит пуш | Команда для загрузки ваших сохранений (коммитов) с компьютера на GitHub. «Отправить на сервер». |
| **git status** (гит ста́тус) | гит ста́тус | Команда для проверки: какие файлы изменены, какие готовы к сохранению. Как «Проверить перед отправкой». |
| **Force push** (форс-пуш) | форс-пуш | Принудительная загрузка на GitHub, которая затирает всё, что там было. ⚠️ Опасно — используйте осторожно! |
| **Ветка** (Branch) | ветка | Параллельная версия проекта. Можно вносить изменения в ветке, не ломая основную версию. Как черновик документа. |
| **main** (мэйн) | мэйн | Основная ветка проекта в Git. Главная версия кода. |
| **Open-source** (опен-сорс) | о́пен-сорс | Открытый код — проекты, исходный код которых доступен всем. Можно изучать, копировать и использовать бесплатно. |

## 🧩 Скиллы и MCP

| Термин | Произношение | Простое объяснение |
|--------|-------------|-------------------|
| **Скилл** (Skill) | скилл | англ. skill (навык) — набор инструкций для ИИ-агента, который обучает его конкретной задаче: дизайн, маркетинг, код-ревью и т.д. Как курс повышения квалификации для ИИ. |
| **FindSkills** (Файнд-Скиллз) | файнд-ски́ллз | Специальный скилл-поисковик. Помогает найти нужные скиллы для вашего проекта. Устанавливается глобально. |
| **MCP** (Эм-Си-Пи) | эм-си-пи́ | англ. Model Context Protocol — «умный API», который позволяет ИИ-агенту напрямую подключаться к внешним сервисам (базы данных, дизайн) без написания кода. |
| **Плагин** (Plugin) | плаги́н | англ. plug in (подключить) — программное расширение для Antigravity с визуальным интерфейсом. Добавляет новые функции в саму программу. В отличие от скиллов, плагинов мало (~50 шт.). |
| **Глобальная установка** | — | Установка скилла/плагина на уровне всей системы — он будет работать во всех проектах. ⚠️ Осторожно: может замедлить систему. |
| **Локальная установка** | — | Установка скилла только в папку конкретного проекта. Работает только в этом проекте. Безопаснее глобальной. |

## 📋 Процессы разработки

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Регламент** | регла́мент | франц. règlement (правило) | Рабочий документ с правилами и знаниями для разработки. Растёт от лекции к лекции. ИИ-агент следует его правилам при работе. |
| **Мердж** (Merge) | мёрдж | англ. merge (объединить) | Объединение двух документов или версий кода в один. Например, объединение старого и нового регламентов. |
| **ТЗ** (Техническое задание) | тэ-зэ́ | русское сокращение | Подробное описание того, что должна делать программа: какие функции, какие ограничения, как должна выглядеть. Инструкция для ИИ. |
| **Тестирование** (Testing) | тести́рование | англ. test (проверка) | Проверка программы: всё ли работает правильно? Все ли кнопки нажимаются? Нет ли ошибок? |
| **Баг-репорт** (Bug report) | баг-репо́рт | англ. bug report (отчёт об ошибке) | Описание найденной ошибки: что произошло, что ожидалось, как воспроизвести. Отправляется разработчику (или ИИ) для исправления. |
| **Бэкап** (Backup) | бэ́кап | англ. backup (резервная копия) | Копия всех данных проекта «на всякий случай». Если что-то сломается — можно восстановить из бэкапа. |
| **Откат** (Rollback) | отка́т | от русского «откатить назад» | Возвращение к предыдущей рабочей версии проекта. Если новые изменения всё сломали — откатываемся. В Antigravity: кнопка **Reject All**. |
| **Декомпозиция** | декомпози́ция | лат. de + compositio (разложение) | Разбивание большой задачи на маленькие подзадачи. Вместо «сделай сайт» → «сделай шапку», «сделай меню», «сделай форму заявки». |
| **Приоритизация** | приоритиза́ция | англ. priority (приоритет) | Расстановка задач по важности: High (высокий), Medium (средний), Low (низкий). Что делать первым, что может подождать. |
| **Транскрибация / Транскрипция** | транскрипция | лат. transcriptio (переписывание) | Превращение аудио- или видеозаписи в текст. Лекцию записали → Whisper-бот превратил в текстовый файл. |
| **Фидбек** (Feedback) | фи́дбек | англ. feedback (обратная связь) | Отзывы пользователей о продукте. «Что нравится? Что неудобно? Чего не хватает?» |
| **Лидогенерация** (Lead generation) | лидогенера́ция | англ. lead (потенциальный клиент) + generation (создание) | Процесс привлечения потенциальных клиентов. Например, через сайт: посетитель оставляет заявку — это лид. |

## 🖥 Общие IT-термины

| Термин | Произношение | Откуда пришёл | Простое объяснение |
|--------|-------------|---------------|-------------------|
| **Гик** (Geek) | гик | англ. geek (знаток, энтузиаст) | Человек, увлечённый технологиями и компьютерами. Это комплимент в IT-мире! |
| **Тильда** (~) | ти́льда | исп. tilde (знак ~) | Волнистый символ на клавиатуре (~). В программировании часто означает «домашняя папка» пользователя. |
| **Директория** (Directory) | директо́рия | англ. directory (каталог) | Папка на компьютере. «Рабочая директория» = папка, в которой сейчас работаете. |
| **Логи** (Logs) | ло́ги | англ. log (журнал) | Записи о том, что делала программа: какие действия выполнила, какие ошибки были. Как бортовой журнал самолёта. |
| **Логирование** | логи́рование | от англ. logging | Процесс записи действий программы в журнал (логи). Помогает понять, что пошло не так. |
| **Кириллица в путях** | — | — | Русские буквы в именах папок и файлов. На Windows часто вызывают ошибки в программах. Решение: называть папки латиницей (английскими буквами). |
| **Контекст** (Context) | конте́кст | лат. contextus (связь) | Информация, которую ИИ «видит» и учитывает при работе. Чем больше контекста (файлы, описания), тем точнее результат. «Контекстное окно» — сколько текста ИИ может «помнить» одновременно. |
| **Фоллбэк** (Fallback) | фо́ллбэк | англ. fall back (отступить) | Запасной вариант. Если основной инструмент не работает — переключаемся на фоллбэк. Например: Claude не отвечает → переключаемся на Gemini. |
| **Accept All / Reject All** | аксе́пт ол / рида́жект ол | англ. accept (принять) / reject (отклонить) | Кнопки в Antigravity. **Accept All** = «Принимаю все изменения». **Reject All** = «Отменяю все изменения» (откат). |
| **Миграция** (Migration) | мигра́ция | лат. migratio (переселение) | Изменение структуры базы данных: добавление новых таблиц, полей, изменение типов данных. Как ремонт: перепланировка, но для данных. |
| **Сервер** (Server) | се́рвер | англ. server (обслуживающий) | Мощный компьютер, который работает 24/7 и хранит ваш сайт/приложение, делая его доступным для пользователей через интернет. |
| **Локально** (Local) | лока́льно | лат. localis (местный) | На вашем собственном компьютере. «Запустить локально» = запустить на своём компьютере, не в интернете. |
| **Маршрутизация** (Routing) | маршрутиза́ция | англ. routing (определение маршрута) | Система, которая определяет, какую страницу или информацию показать пользователю в зависимости от его запроса. Как диспетчер: направляет запрос туда, куда нужно. |
| **Human-in-the-Loop** | хью́ман-ин-зе-лу́п | англ. (человек в цикле) | Принцип работы с ИИ, при котором все важные решения одобряет человек. ИИ предлагает — человек проверяет и подтверждает. |
| **Staging area** | стэ́йджинг э́рия | англ. (промежуточная зона) | В Git: «зал ожидания» для файлов. Файлы, добавленные через `git add`, попадают сюда и ждут, пока вы сделаете коммит. |
| **E-commerce** (И-коммерс) | и-ко́ммерс | англ. electronic commerce (электронная торговля) | Покупка и продажа товаров через интернет. Интернет-магазины — это e-commerce. |
| **SEO** (Эс-И-О) | эс-и-о́ | англ. Search Engine Optimization | Оптимизация сайта для поисковых систем. Чтобы ваш сайт находили в Google и Яндекс по нужным запросам. |
| **VPN** (Ви-Пи-Эн) | ви-пи-э́н | англ. Virtual Private Network | Виртуальная частная сеть — программа, которая позволяет заходить на заблокированные сайты и защищает ваш интернет-трафик. |
| **Markdown** (.md) | ма́рк-даун | англ. mark down (разметить) | Простой способ форматирования текста с помощью символов: `#` — заголовок, `**текст**` — жирный, `-` — список. Файлы с расширением `.md`. Используется во всех документах проекта. |
| **PowerShell** | пауэр-шелл | англ. power (мощь) + shell (оболочка) | Программа-терминал в Windows для ввода команд текстом. Более мощная версия обычной командной строки. |
| **Drag & Drop** | драг-энд-дроп | англ. drag (тащить) + drop (бросить) | «Перетащи и брось» — способ перемещения файлов мышкой. В Antigravity: перетаскиваете файл в чат для добавления контекста. |
| **A/B-тестирование** | эй-би-тести́рование | англ. A/B testing | Метод сравнения двух вариантов (вариант А и вариант Б), чтобы понять, какой работает лучше. Skill Creator делает это автоматически для скиллов. |
| **TTS** (Ти-Ти-Эс) | ти-ти-э́с | англ. Text-to-Speech (Текст в речь) | Технология, которая превращает текст обратно в голос. Нейросеть читает написанный текст вслух (чтобы бот мог разговаривать с вами). |
