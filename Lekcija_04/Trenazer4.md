## Блок 1: Понимание концепций

### Вопрос 1
**Чем Claude Code принципиально отличается от Antigravity?**

<details>
<summary>✅ Ответ</summary>

- Antigravity — готовая IDE с визуальным интерфейсом и кнопкой Run
- Claude Code — терминальный агент, более гибкий, но без визуального контроля
- Оба работают на Opus 4.6, но Claude Code позволяет бесконечно кастомизировать скиллами и субагентами
- Claude Code может выходить за рамки разработки (маркетинг, контент, аналитика)
</details>

---

### Вопрос 2
**Почему в Claude Code нужно быть осторожнее с запросами, чем в Antigravity?**

<details>
<summary>✅ Ответ</summary>

- В Antigravity есть кнопка **Reject** — легко откатить изменения
- Claude Code **безвозвратно** меняет файлы
- Если написать «объясни мне это» без уточнения «не пиши код» — он может начать менять файлы
- Неточная задача → потраченные токены и сломанный проект
</details>

---

### Вопрос 3
**В чём разница между скиллом и агентом (субагентом)?**

<details>
<summary>✅ Ответ</summary>

| | Скилл | Агент |
|---|---|---|
| Суть | Текстовые инструкции (.md) | Контейнер с инструментами |
| Содержит | Текст, ссылки | Инструкции + MCP + API |
| Аналогия | Инструкция | Набор инструментов |
</details>

---

### Вопрос 4
**Когда скилл нужно ставить глобально, а когда локально?**

<details>
<summary>✅ Ответ</summary>

- **Глобально (~/)** — универсальные скиллы: аудит кода, безопасность, покрытие кода
- **Локально (./)** — специфичные скиллы: дизайн для конкретного проекта, контент для определённого сайта
- Если специфика проектов сильно меняется — локальные. Если скилл нужен везде — глобальный
</details>

## Блок 2: Практические задания

### Упражнение 1: Первый запрос в Claude Code
Напишите запрос для Claude Code, чтобы он **изучил ваш проект и дал оценку**. Не забудьте:
- Указать, что именно оценивать
- Разбить на подзадачи

<details>
<summary>💡 Пример</summary>

```
Изучи весь проект. Запусти 3 агента:
1. Найди потенциальные баги и ошибки
2. Оцени качество структуры кода
3. Дай рекомендации по улучшению
Результаты предоставь в виде сводной таблицы.
```
</details>

---

### Упражнение 2: Создание скилла
Вы хотите, чтобы Claude Code всегда **проверял безопасность** при установке новых скиллов. Как бы вы сформулировали задачу для Skill Creator?

<details>
<summary>💡 Пример</summary>

```
/skill-creator

Создай скилл "security-check", который:
1. Перед установкой любого нового скилла проверяет его содержимое
2. Ищет подозрительные команды (удаление файлов, отправка данных)
3. Оценивает репутацию источника
4. Выдаёт отчёт: безопасно / подозрительно / опасно
```
</details>

---

### Упражнение 3: Сценарий «Аудитор»
Опишите пошаговый процесс: как использовать Claude Code для проверки работы Antigravity.

<details>
<summary>💡 Ответ</summary>

1. Antigravity выполняет задачу (например, «Добавь форму регистрации»)
2. Переключаемся в терминал Claude Code
3. Пишем: «Используя скилл review, проверь последние изменения в проекте»
4. Claude Code анализирует, находит ошибки
5. Копируем отчёт → вставляем в Antigravity
6. Antigravity исправляет по конкретным замечаниям
</details>

## Блок 3: Вопросы на закрепление

### Мини-тест

1. **Что такое MCP?**
   - а) Язык программирования
   - б) Метод подключения к внешним сервисам ✅
   - в) Модель нейросети

2. **Что делает Context7?**
   - а) Хранит пароли
   - б) Обеспечивает актуальную документацию для агента ✅
   - в) Переводит тексты

3. **Что произойдёт, если Opus 5 научится делать дизайн идеально?**
   - а) Все скиллы перестанут работать
   - б) Дизайнерские скиллы автоматически отпадут ✅
   - в) Нужно будет обновить все скиллы вручную

4. **Skill Creator создаёт:**
   - а) 1 скилл и проверяет его
   - б) 5 скиллов, тестирует и выбирает лучший ✅
   - в) 10 скиллов без тестирования

5. **Какой тариф Claude Code рекомендуется для полноценной работы?**
   - а) Pro ($20) — хватает надолго
   - б) Max ($100) — без ограничений ✅
   - в) Бесплатный — достаточно
