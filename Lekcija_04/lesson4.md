## 1. Что такое Claude Code (Код-Код / Cloud Cloud)

### 1.1 Определение
- **Claude Code** — это продукт компании Anthropic, работающий из терминала
- Разворачивается **локально** на компьютере, запускается из терминала внутри Antigravity
- Работает на модели **Opus 4.6** — той же, что и Antigravity
- Аналог OpenClaw, но **безопаснее** и с активным комьюнити

### 1.2 Чем отличается от Antigravity

| Критерий | Antigravity | Claude Code |
|---|---|---|
| **Тип** | Готовая среда разработки (IDE-агент) | Терминальный агент |
| **Модель** | Opus 4.6 | Opus 4.6 |
| **Интерфейс** | Визуальный чат + кнопка Run | Терминал (текстовые команды) |
| **Скиллы** | Иногда игнорирует | Автоматически подхватывает |
| **Субагенты** | Есть (Open Agent Manager) | Полноценная многоагентная система |
| **Откат изменений** | Кнопка Reject — легко | Нет простого отката — нужна аккуратность |
| **Цена** | Подписка (~фикс), экономичнее | Подписка Pro ~$20, Max ~$100 |
| **Кому подойдёт** | Новички, стабильная разработка | Продвинутые, гибкая настройка |

### 1.3 Зачем нужен Claude Code, если есть Antigravity?
1. **Нет проблем с терминалом** — не нужно нажимать Run
2. **Многоагентная система** — параллельная работа 2–15 агентов
3. **Бесконечная кастомизация** — скиллы, субагенты, MCP
4. **Выход за рамки разработки** — маркетинг, контент, аналитика
5. **Работа с телефона** (через Telegram) — удалённое управление
6. **Личный ассистент** — можно превратить в «Джарвиса»

> ⚠️ **Важно:** При работе с Claude Code нужно **чётко формулировать задачи**. Иначе он может потратить токены впустую или сломать проект без возможности отката.

## 2. Установка Claude Code

### 2.1 Пошаговая установка

```powershell
# 1. Установить Node.js (если ещё нет)
# Скачать нужную версию с nodejs.org

# 2. Установить Claude Code глобально
npm install -g @anthropic-ai/claude-code

# 3. Запустить Claude Code
claude
```

### 2.2 Авторизация
- При первом запуске выбрать **вариант 1** — подписка (безлимит)
- Вариант 2 (API-биллинг) — платно за токены, **не рекомендуется** для начинающих
- Авторизоваться на том аккаунте, **где оплачена подписка Claude**

### 2.3 Удобное расположение
- Правой кнопкой по панели терминала → **Move Panel → Right**
- Так Claude Code будет рядом с чатом Antigravity

### 2.4 Переключение модели
- В Claude Code набрать `/model`
- Стрелками выбрать **Opus 4.6** (по умолчанию может стоять Sonnet)

## 3. Многоагентная система (Agents Team)

### 3.1 Что это
- Можно запустить **несколько агентов параллельно** над одним проектом
- Каждый агент работает **в своей песочнице** — невидимо
- Результаты приходят в общий чат

### 3.2 Пример запроса на 5 агентов

```
Я хочу, чтобы ты в моём проекте запустил 5 агентов:
1. Изучи код и найди потенциальные баги
2. Изучи код и оцени процент покрытия кода
3. Опиши функционал проекта
4. Проверь, что нужно ещё доработать
5. Сделай оценку проекта как full-stack разработчик
```

### 3.3 Принцип «Начальника отдела»
- Можно создать **главного агента** (например, «Анатолий»)
- У него в подчинении: охотник за багами, аналитик, документатор, ревьюер, оценщик
- Даёте одну команду → вся команда работает самостоятельно

> 💡 Количество агентов зависит от мощности вашего компьютера.

## 4. Скиллы (Skills) — версия 2.1

### 4.1 Что такое скиллы
- **Скилл** = набор текстовых инструкций в `.md` файле
- Описывают пошаговую последовательность действий
- Улучшают базовые способности Claude Code

### 4.2 Два уровня скиллов

| Уровень | Описание | Пример |
|---|---|---|
| **1-й уровень** | Улучшение того, что Claude Code и так умеет | Скилл для лучшего дизайна |
| **2-й уровень** | Самообучающийся скилл, улучшается с каждым использованием | Скилл для КП, который учится на ваших правках |

### 4.3 Умная система подключения скиллов
- Claude Code **сам решает**, нужен ли скилл для задачи
- Если справляется без скилла — **не подключает** его
- Неиспользуемые скиллы **автоматически удаляются**
- При выходе Opus 4.7/5.0 — многие скиллы отпадут сами

### 4.4 Глобальные vs Локальные скиллы

| Тип | Путь | Когда использовать |
|---|---|---|
| **Глобальный** | `~/.claude/skills/...` (тильда) | Универсальные: аудит кода, безопасность |
| **Локальный** | `.claude/skills/...` (в проекте) | Специфичные: дизайн для контент-завода |

> 💡 **Правило:** Если специфика работы сильно меняется между проектами — используйте **локальные** скиллы. Глобально ставьте только **универсальные**.

### 4.5 Версионирование скиллов
- Каждое улучшение = новая версия (1.0 → 1.1 → 1.2)
- Обратная связь от вас → скилл обновляется → работает лучше
- На ~20-й итерации — скилл работает **идеально**

## 5. Skill Creator — создание скиллов

### 5.1 Что делает
- Создаёт сразу **5 вариантов** скилла
- **Тестирует** каждый вариант
- **Сравнивает** с работой чистого Claude Code (без скилла)
- Выбирает **лучший** и согласует с вами
- Это внутренний **A/B-тест**

### 5.2 Установка Skill Creator

```
# В Claude Code набрать:
/plugins

# Перейти в Discover → поиск → "skill"
# Выбрать Skill Creator
# Установить ГЛОБАЛЬНО (User scope)
# Перезапустить терминал!
```

### 5.3 Использование

```
# В Claude Code:
/skill-creator

# Описать задачу, например:
"Создай скилл, который будет качественно делать проверку кода"
```

## 6. Скиллы vs Агенты (субагенты)

| Критерий | Скиллы | Агенты (субагенты) |
|---|---|---|
| **Суть** | Текстовые инструкции (.md) | Контейнер с инструментами |
| **Содержимое** | Текст, ссылки, пояснения | Инструкции + MCP + API-коннекторы |
| **Аналогия** | Инструкция «как чинить» | Готовый набор инструментов |
| **Пример** | «Делай ревью по 10 правилам» | Агент аналитики с подключением к метрикам |

## 7. MCP-серверы (Model Context Protocol)

### 7.1 Что это
- Метод **быстрого подключения** к внешним сервисам
- Никакой магии — просто стандартизированный протокол

### 7.2 Пример: Context7
- Собрана **вся актуальная документация** популярных библиотек
- Агент **автоматически проверяет** документацию перед написанием кода
- При обновлении API — код всегда соответствует последней версии

### 7.3 Пример: GitHub MCP
- Публикация проекта за **секунды** (в 2 раза быстрее, чем через API)

## 8. Голосовой ввод — Gladia

- Сторонний сервис для голосового ввода текста
- Вызывается **горячей клавишей** → говорите → текст появляется в поле ввода
- Работает и в Antigravity, и в Claude Code
- Подписка ~$20/мес, есть бесплатный период
- Альтернатива: можно **самому разработать** аналог

## 9. Whisper и TTS — распознавание речи

### 9.1 Бесплатные варианты
- **Groq** — лучший бесплатный Whisper (требует VPN)
- **Яндекс SpeechKit** — хороший, но дорогой (~40-60 коп.)
- **OpenRouter** — можно искать TTS-модели

### 9.2 Рекомендация
- Основная модель (генерация текста) → **OpenRouter**
- TTS/Whisper → ключ от **ChatGPT** (экономичнее)

## 12. Практические сценарии использования

### 12.1 Claude Code как аудитор
1. Разрабатываете проект в **Antigravity**
2. Переключаетесь в **Claude Code** → «проверь код проекта»
3. Получаете обратную связь
4. Копируете результат → вставляете в **Antigravity** → исправляет

### 12.2 Claude Code как личный ассистент
- Подключение к **Kanban-доске** → автоматическое выполнение задач
- **Уведомления в Telegram** о выполненных задачах
- Работа в автономном режиме (пока вы не за компьютером)

### 12.3 Два агента = контроль качества
- **Antigravity** — разрабатывает (задача 1, 2, 3...)
- **Claude Code** — проверяет каждую задачу с нужными скиллами
- Результат проверки → обратно в Antigravity для правок

## 13. Важные предупреждения

> ⚠️ **Не используйте Claude Code как чат!** Если хотите просто спросить — пишите: «Не пиши код, объясни мне...». Иначе он может начать менять файлы.

> ⚠️ **Нет кнопки Reject!** В отличие от Antigravity, Claude Code **безвозвратно** меняет файлы. Работайте аккуратно.

> ⚠️ **Лимиты подписки.** На тарифе Pro ($20) лимиты быстро заканчиваются. Для полноценной работы рекомендуется Max ($100).

## 14. Ключевые выводы

1. **Claude Code** — мощный терминальный агент для продвинутой работы
2. **Antigravity** остаётся лучшим выбором для стабильной разработки
3. **Оба инструмента дополняют друг друга** — используйте вместе
4. **Скиллы** = текстовые инструкции, **агенты** = контейнеры с инструментами
5. **Skill Creator** помогает создавать и тестировать скиллы автоматически
6. **MCP-серверы** обеспечивают актуальность документации и быстрые подключения
7. **Многоагентная система** — главное преимущество Claude Code
8. **Голосовой ввод** (Gladia) работает в обоих инструментах
