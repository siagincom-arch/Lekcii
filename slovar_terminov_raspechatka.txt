СЛОВАРЬ ТЕРМИНОВ — VIBECODING (ЛЕКЦИИ 1–4)
Для распечатки

========================================================================
1. ОСНОВНЫЕ ТЕРМИНЫ
========================================================================

Vibecoding (Вайбкодинг)
[От: англ. vibe (атмосфера) + coding (написание кода)]
Метод создания программ и приложений, где человек описывает свою идею словами, а искусственный интеллект сам пишет код. Как если бы вы объясняли строителю, что хотите, а он сам строит.

AI / ИИ (Эй-Ай)
[От: англ. Artificial Intelligence]
Искусственный интеллект — компьютерная программа, которая умеет «думать»: анализировать текст, писать код, отвечать на вопросы. Например, ChatGPT или Claude.

Агент (Agent)
[От: англ. agent (действующее лицо)]
Программа-помощник на основе ИИ, которая сама выполняет задачи: ищет информацию, пишет код, запускает команды. Вы даёте ей задание — она выполняет.

Субагент (Subagent)
[От: англ. sub (под) + agent]
Вспомогательный агент. Основной агент может поручить часть работы субагенту. Как начальник делегирует задачу помощнику. В Antigravity субагенты = скиллы.

Промпт (Prompt)
[От: англ. prompt (подсказка, запрос)]
Текстовый запрос, который вы пишете искусственному интеллекту. Например: «Сделай мне сайт для кафе» — это промпт. Чем точнее промпт, тем лучше результат.


========================================================================
2. ИНСТРУМЕНТЫ И ПЛАТФОРМЫ
========================================================================

Antigravity (Антигравити)
Программа для разработки (IDE). Похожа на текстовый редактор, но со встроенным ИИ-помощником. Сделана на базе VS Code.

IDE (Ай-Ди-И)
«Рабочий стол программиста» — программа, в которой пишут код.

VS Code (Ви-Эс Код)
Популярная бесплатная программа для написания кода от Microsoft.

Claude / Claude Opus (Клод)
Искусственный интеллект, который умеет писать код и вести диалог. Основной «мозг» в Antigravity.

Gemini (Джемини)
ИИ от Google. Используется как запасной вариант. Хорошо работает с большими файлами.

LLM (Эл-Эл-Эм)
Большая языковая модель — тип ИИ (Claude, Gemini, ChatGPT).

OpenRouter (ОпенРоутер)
Сервис-посредник для подключения к разным ИИ-моделям через один вход.

MiniMax (МиниМакс)
Модель ИИ для автоматических быстрых задач. Быстрее и дешевле Claude.

Supabase (Супабейс)
Онлайн-сервис для хранения данных (база данных). Как мощная таблица Excel в интернете.

GitHub (ГитХаб)
Сайт-хранилище для кода. Как облачный диск, но специально для программистов.

Git (Гит)
Программа (система контроля версий), которая запоминает все изменения в файлах. «Машина времени» для кода.

Whisper-bot (Виспер-бот)
Программа, которая превращает аудио и видео в готовый текст.

n8n (Эн-Эйт-Эн)
Программа для автоматизации процессов (соединяет разные сервисы без кода).

Perplexity (Перплексити)
Поисковик на основе ИИ. Ищет информацию и даёт готовые ответы с источниками.

Context7 (Контекст7)
Сервис, который помогает ИИ быстро находить документацию по программам.

Figma (Фигма)
Программа для создания дизайна сайтов и приложений в браузере.

OpenClaw / Clobot (Опен-кло)
Автономный агент-валидатор (контролёр качества), бесплатная альтернатива Claude Code.

Claude Code (Код-Код)
Терминальная программа-агент от Anthropic, запускающая команды без кнопок отмены.

Skill Creator (Скилл криэйтор)
Инструмент для создания и A/B-тестирования скиллов.

Gladia (Гладия)
Встроенный сервис для безупречного голосового ввода.

Groq / Whisper
Системы распознавания речи (голос в текст).


========================================================================
3. ВЕБ-РАЗРАБОТКА
========================================================================

Фронтенд (Frontend)
Всё, что видит пользователь на экране (дизайн, кнопки, тексты). «Лицо» сайта.

Бэкенд (Backend)
Невидимая часть: серверы, базы данных, логика. «Мозг» сайта.

Лендинг (Landing page)
Одностраничный сайт для конкретной цели (например, продать товар или собрать контакты).

UI (Ю-Ай / User Interface)
Внешний вид приложения: кнопки, меню.

UX (Ю-Экс / User Experience)
Удобство использования приложения для пользователя.

URL (Ю-Ар-Эл)
Адрес страницы в интернете (например: https://google.com).

Next.js (Некст-Джей-Эс)
Инструмент для создания красивых веб-приложений.

Фреймворк (Framework)
Набор готовых инструментов (каркас) для создания программ. Как конструктор LEGO.

Хостинг (Hosting)
Сервис, на котором размещается ваш сайт, чтобы он был доступен в интернете.

Vercel (Версель) / Railway (Рэйлвей) / VPS (Ви-Пи-Эс)
Сервисы и виртуальные серверы для размещения (хостинга) приложений.


========================================================================
4. ПРОГРАММИРОВАНИЕ (ОБЩИЕ ТЕРМИНЫ)
========================================================================

Стек (Stack)
Набор технологий и инструментов, используемых в проекте.

Код (Code)
Текст из команд, понятных компьютеру. 

Кодогенерация
Автоматическое создание кода с помощью ИИ.

Скрипт (Script)
Небольшая программа для конкретной задачи.

Баг (Bug)
Ошибка (поломка) в программе.

Дебаг / Дебаггинг (Debug)
Процесс поиска и исправления багов. Починка кода.

Фикс (Fix)
Исправление бага («зафиксить» = исправить).

Итерация (Iteration)
Один цикл улучшения продукта (сделали, проверили, улучшили).

Деплой (Deploy) / Продакшн (Production) / Релиз (Release)
Размещение программы на сервере и её выпуск для реальных пользователей.

Прототип (Prototype)
Первая пробная версия продукта без финального дизайна.

API (Эй-Пи-Ай)
Способ взаимодействия программ между собой (как передача заказа на кухню ресторана).

JSON (Джейсон)
Удобный текстовый формат хранения данных (как структурированная анкета).

SQL (Эс-Кью-Эл)
Язык для запросов к базам данных (как умный поиск в таблицах).

Терминал (Terminal) / PowerShell
Текстовое окно для ввода системных команд (вместо мыши используются текстовые команды).

npm (Эн-Пи-Эм)
Менеджер пакетов. Инструмент для установки дополнительных программ к проекту.

Токен (Token) / Переменные окружения (.env)
Секретные ключи и пароли приложения. Хранятся локально в скрытом файле .env.

Эндпоинт (Endpoint)
Конкретный адрес (URL), через который программа принимает запросы.

Edge-кейс (Edge case)
Нестандартная, крайняя ситуация или ошибка, которую надо предусмотреть.


========================================================================
5. ПРОЦЕССЫ И GIT
========================================================================

Репозиторий (Repo)
Папка с проектом на сайте GitHub. Хранит историю изменений.

git clone / git add / git commit / git push / git status
Консольные команды для скачивания, сохранения и отправки проекта на GitHub.

Коммит (Commit)
Одно «сохранение» важных изменений проекта с текстовым описанием.

main (мэйн) / Ветка (Branch)
«Ветка» — копия проекта для экспериментов. «main» — главная рабочая версия кода.

Force push
Опасная команда для принудительной загрузки на GitHub (стирает старое).

Open-source
Общедоступный, открытый код, бесплатный для изучения.

Скилл (Skill) / Плагин
Конкретный навык, которому обучен ИИ (скилл — в виде текста, плагин — с интерфейсом). 

FindSkills / MCP
Инструменты для поиска скиллов и протокол для подключения ИИ к сторонним данным без кода.

Глобальная / Локальная установка
Глобальная — для всех проектов (на весь ПК). Локальная — только в текущую папку проекта.

Регламент
Рабочий документ с правилами разработки (наша "библия" процессов).

Мердж (Merge)
Объединение (слияние) двух версий документа или кода в один.

ТЗ / Декомпозиция
Техническое задание и его разбивка на мелкие, решаемые подзадачи.

Тестирование / Баг-репорт
Проверка программы и письменный отчет об обнаруженных ошибках.

Бэкап / Откат (Rollback)
Создание резервной копии и возвращение к старой версии (в Antigravity: Reject All).

Транскрибация
Перевод аудио в текст.

Фидбек / Лидогенерация
Обратная связь от пользователей / Привлечение потенциальных клиентов.


========================================================================
6. 10 КРАТКИХ СОВЕТОВ РАЗРАБОТЧИКУ
========================================================================
1. Гик (Geek) — знаток технологий (это комплимент).
2. Тильда (~) — символ вашей домашней папки на компьютере.
3. Логи (Logs) — бортовой журнал программы, там ищем ошибки.
4. Избегайте кириллицы в путях — папки называем только английскими буквами.
5. Фоллбэк (Fallback) — запасной план, если что-то легло.
6. Accept All / Reject All — главные кнопки принятия кода и отката назад в Antigravity.
7. Миграция — изменение структуры базы данных (перепланировка).
8. Маршрутизация (Routing) — логика переходов между страницами на сайте.
9. Drag & Drop — просто мышкой тащите файлы в чат, чтобы учить агента новым трюкам!
10. Human-in-the-Loop — главный закон: ИИ предлагает, а вы (человек) одобряете.
