## 1. Что такое Vibecoding

**Vibecoding** — методология скоростного создания цифровых продуктов, где разработчик описывает цели и структуру («вайб»), а AI-агент берёт на себя написание кода, тестирование и деплой.

**Ключевые принципы:**
- Разработчик = пилот, AI = инструмент. Финальный контроль всегда у человека.
- Скорость: путь от идеи до рабочего прототипа — часы, а не недели.
- Структурированный подход через фиксированные этапы разработки.

## 2. Стек инструментов (экосистема «Antigravity»)

### Основные инструменты

| Инструмент | Назначение |
|---|---|
| **Antigravity** | AI-IDE (среда разработки на базе VS Code) |
| **Claude Opus** | Основная LLM — логика, кодогенерация, сложные рассуждения |
| **Gemini 3.1 Pro** | Резервная LLM — быстрая, большое контекстное окно, анализ объёмных файлов |
| **OpenRouter** | Прокси-оркестратор для доступа к разным моделям через один API |
| **MiniMax 2.5** | Модель для агентских задач (через OpenRouter). Полцены DeepSeek, качество уровня Opus |
| **OpenClaw (Clobot)** | Автономный агент валидации — мониторит работу, генерирует отчёты |

### Инфраструктура

| Инструмент | Роль |
|---|---|
| **Supabase** | Бэкенд/БД (замена Google Sheets / Airtable) |
| **GitHub** | Контроль версий + библиотека готовых решений |
| **Whisper-bot** | Транскрипция аудио/видео в текст для обработки AI |
| **n8n** | Автоматизация сложных рабочих процессов |

### Стратегия использования моделей
- **По умолчанию** → Claude Opus (для логики и кода)
- **Фоллбэк** → Gemini (если Opus/Claude зависает, особенно на Windows)
- **Агентские задачи** → MiniMax 2.5 (эффективнее и дешевле)
- **Глубокие исследования** → Gemini 3.1 (быстрый анализ больших объёмов)

## 3. Жизненный цикл продукта — 11 этапов (0–10)

Цикл **итеративный**: этап 10 → возврат к этапу 0 для следующей итерации.

| # | Этап | Суть |
|---|---|---|
| **0** | Исследование | AI-сбор данных, анализ рынка, проверка осуществимости |
| **1** | Идея | Концепция, бизнес-ценность, «вайб» продукта |
| **2** | Техническое задание (ТЗ) | Перевод идеи в инструкции для AI: фичи, ограничения, требования |
| **3** | Архитектура проекта | Структура каталогов, модели данных, связи компонентов |
| **4** | Требования к дизайну | UI/UX, визуал, взаимодействия (включая ботов: кнопки, эмодзи) |
| **5** | Локальный запуск | Генерация и запуск кода в локальной среде |
| **5.1** | Задачи | Декомпозиция на конкретные задачи, приоритизация (High/Medium/Low) |
| **6** | Тестирование | Поиск багов, edge-кейсов; AI-верификация по ТЗ |
| **7** | Деплой | Вывод на продакшн-сервер, настройка окружения |
| **8** | Дебаггинг | Фикс конкретных багов после деплоя |
| **9** | Релиз | Финальный запуск для пользователей |
| **10** | Пост-релизное исследование | Аналитика, фидбек, планирование следующей итерации → Этап 0 |

## 4. Безопасность при работе с AI (10 правил)

1. **AI не автономен** — действует только по команде разработчика
2. **Изоляция директорий** — работать только в папке проекта
3. **Запрет глобального удаления файлов** — ограниченная среда
4. **Использовать production-серверы** — изолировать от основной машины
5. **Глобальный промпт** — явные правила, что AI может и не может делать
6. **Логирование действий** — читать логи перед одобрением изменений
7. **Виртуальные машины** — для рискованных экспериментов
8. **Автоматические бэкапы** — частые снапшоты, GitHub для истории
9. **Защита секретов** — ключи в `.env`, исключённом из git
10. **Human-in-the-Loop** — пошаговое одобрение до стабилизации проекта

## 7. Ключевые выводы

- **Vibecoding ≠ «no-code»** — это структурированная методология с чёткими этапами.
- **Коммерческая ценность** — участники уже зарабатывают (автоматизация магазинов, лидогенерация).
- **Разнообразие применений** — от медицины до мебели и духовных практик.
- **AI — ускоритель**, но контроль и безопасность остаются ответственностью человека.
