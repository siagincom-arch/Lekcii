# 🏋️ Тренажёр Vibecoding — Пошаговое руководство по созданию продукта

> Используй этот тренажёр каждый раз, когда начинаешь новый проект. Проходи этапы последовательно, заполняя каждый блок. Не перескакивай — качество результата зависит от дисциплины на ранних этапах.

---

## 🔄 Общий цикл разработки

```
0. Исследование → 1. Идея → 2. ТЗ → 3. Архитектура → 4. Дизайн
→ 5. Локальный запуск → 5.1 Задачи → 6. Тестирование
→ 7. Деплой → 8. Дебаггинг → 9. Релиз → 10. Пост-релиз исследование
                                                        ↓
                                              Возврат к этапу 0
```

---

## Этап 0. Исследование 🔍

**Цель:** Понять рынок, конкурентов, осуществимость идеи.

### Что делать:
1. Открой новый чат в Antigravity

3. Изучи результат, задай уточняющие вопросы

### Чеклист:
- [ ] Конкуренты и аналоги изучены
- [ ] Целевая аудитория определена
- [ ] Осуществимость подтверждена
- [ ] Результат сохранён в `0_исследование.md`

```

```

## Этап 1. Идея 💡

**Цель:** Описать идею максимально подробно своими словами. Чем детальнее — тем лучше результат.

### Что делать:
1. Опиши идею **своими словами** — не стесняйся, пиши как думаешь
2. Важно ответить на вопросы:
   - Какую **проблему** решает продукт?
   - **Кто** будет пользоваться?
   - Какие **входные данные** нужны?
   - Какой **результат** (выход) ожидается?
   - В чём **уникальность**?
3. Отправь описание в Antigravity: *«Изучи мою идею и добавь её в файл идеи»*
4. Прикрепи файл `1_идея.md`

### Промпт для Antigravity:
```
Изучи мою идею перед разработкой и добавь её в файл идеи.
Структурируй: проблема, решение, целевая аудитория, 
входные/выходные данные, уникальность.
```

### Чеклист:
- [ ] Идея описана своими словами
- [ ] Проблема чётко сформулирована
- [ ] Целевая аудитория определена
- [ ] Входные и выходные данные описаны
- [ ] Нейронка структурировала идею
- [ ] Результат проверен и скорректирован
- [ ] Сохранено в `1_идея.md` → нажал **Accept All**

### Заметки:
```
(запиши сюда свою идею черновиком)


```

## Этап 2. Техническое задание 📋

**Цель:** Превратить идею в детальные инструкции для нейронки. ТЗ = скелет продукта.

### Что делать:
1. Прикрепи в чат файл `1_идея.md`
2. Прикрепи пустой файл `2_техническое_задание.md`
3. Напиши промпт (ниже)
4. Проверь результат — это ТЗ больше для нейронки, но пробегись глазами

### Промпт для Antigravity:
```
На основе моей идеи [прикрепить 1_идея.md] 
сформируй техническое задание [прикрепить 2_техническое_задание.md]
```

### Чеклист:
- [ ] ТЗ сгенерировано
- [ ] Функциональные требования перечислены
- [ ] Технологический стек определён
- [ ] Результат проверен
- [ ] Сохранено в `2_техническое_задание.md` → **Accept All**

### Если что-то не так:
- Мелкие правки → правь прямо в файле
- Глобальные проблемы → напиши в чат, что исправить
- Всё плохо → **Reject All** и переформулируй запрос

## Этап 3. Архитектура проекта 🏗️

**Цель:** Создать файловую структуру (директорию) будущего кода. Без порядка = хаос = плохой результат.

### Что делать:
1. Прикрепи в чат `1_идея.md` + `2_техническое_задание.md`
2. Попроси сформировать архитектуру

### Промпт для Antigravity:
```
На основе моей идеи [прикрепить] и технического задания [прикрепить]
сформируй архитектуру проекта — файловую структуру и компоненты.
Запиши в [прикрепить 3_архитектура_проекта.md]
```

### Чеклист:
- [ ] Файловая структура создана
- [ ] Компоненты системы описаны
- [ ] Модель данных определена (если есть БД)
- [ ] API эндпоинты описаны (если есть)
- [ ] Сохранено в `3_архитектура_проекта.md` → **Accept All**

## Этап 4. Требования к дизайну 🎨

**Цель:** Определить внешний вид продукта. Дизайн есть у всего — даже у Telegram-бота (кнопки, эмодзи, тексты).

### Что делать:
1. Прикрепи в чат все предыдущие файлы (идея + ТЗ + архитектура)
2. Опиши, как хочешь, чтобы продукт выглядел

### Промпт для Antigravity:
```
На основе моей идеи, ТЗ и архитектуры 
сформируй требования к дизайну продукта.
Включи: цветовую палитру, стиль, компоненты интерфейса.
Если это бот — опиши тексты кнопок, эмодзи, структуру меню.
Запиши в 4_требования_к_дизайну.md
```

### Чеклист:
- [ ] UI/UX концепция описана
- [ ] Цветовая палитра выбрана
- [ ] Компоненты интерфейса определены
- [ ] Сохранено в `4_требования_к_дизайну.md` → **Accept All**

## Этап 5. Запуск продукта локально 🚀

**Цель:** Сгенерировать код и запустить продукт на своём компьютере.

### Что делать:
1. Прикрепи в чат ТЗ + архитектуру + дизайн
2. Попроси разработать и запустить

### Промпт для Antigravity:
```
На основе технического задания, архитектуры и требований к дизайну
разработай продукт и запусти его локально.
```

### Чеклист:
- [ ] Код сгенерирован
- [ ] Продукт запускается без ошибок
- [ ] Базовая функциональность работает
- [ ] Сохранено в `5_запуск_продукта.md`

### ⚠️ Если зависает (Windows):
1. Нажми **Стоп** и попроси продолжить
2. Переключи модель: Gemini → **Claude Opus**
3. Если совсем не идёт — выводи команды в терминал вручную

## Этап 6. Тестирование 🧪

**Цель:** Проверить, что всё работает корректно. Найти косяки до деплоя.

### Что делать:
1. Протестируй каждую функцию из ТЗ
2. Записывай баги и несоответствия
3. Отправляй баг-репорты в чат Antigravity для исправления

### Правило тестирования:
> **Работает продукт или не работает — вот единственный критерий.** 
> Не лезь в код, не оценивай его качество. Важен результат.

### Чеклист:
- [ ] Все функции протестированы
- [ ] Баги записаны и исправлены
- [ ] Повторное тестирование пройдено
- [ ] Если сломалось → **Reject All** → откат
- [ ] Сохранено в `6_тестирование.md`

---

## Этап 7. Деплой 🌐

**Цель:** Разместить продукт на сервере, чтобы он работал не только на твоём компе.

### Что делать:
1. Выбери хостинг (Vercel, Railway, VPS)
2. Настрой переменные окружения
3. Подключи базу данных (Supabase)
4. Залей код

### Промпт для Antigravity:
```
Задеплой текущий проект на [платформа].
Настрой переменные окружения и подключение к Supabase.
```

### Чеклист:
- [ ] Хостинг выбран
- [ ] Проект задеплоен
- [ ] Переменные окружения настроены
- [ ] Продукт доступен по URL
- [ ] Сохранено в `7_деплой.md`

**Цель:** Исправить конкретные баги, найденные после деплоя.

### Что делать:

2. Записывай каждый баг
3. Отправляй описание бага в Antigravity для фикса

### Чеклист:

## 📌 Шпаргалка: быстрые команды

| Действие | Что делать |
|---|---|
| **Подтвердить изменения** | Нажать **Accept All** (синяя кнопка) |
| **Откатить изменения** | Нажать **Reject All** |
| **Новый чат** | Нажать **+** (справа) |
| **Посмотреть историю** | Кнопка с кружком-стрелкой → Show More |
| **Прикрепить файл** | Перетащить файл в чат |
| **Прикрепить папку** | Перетащить всю папку в чат (= полный контекст) |

## 🛡️ 10 правил безопасности

1. AI не действует автономно — ты контролируешь
2. Работай только в папке проекта (изоляция)
3. Не разрешай глобальное удаление файлов
4. Для рискованных задач — используй отдельный сервер/ВМ
5. Пропиши глобальный промпт (правила для AI)
6. Проси логировать все действия
7. Смотри внимательно, что запускается в терминале
8. Делай бэкапы (GitHub + копии)
9. Храни ключи в `.env`, исключённом из git
10. Пошагово одобряй изменения до стабилизации

---

## 🧠 Полезные советы из лекции

- **Модель по умолчанию** → Claude Opus. Фоллбэк → Gemini 3.1
- **GitHub = библиотека** — ищи готовые решения, а не пиши с нуля
- **Whisper-бот** — транскрибируй все разговоры с клиентами
- **Правило №1** — работает продукт или нет, остальное неважно
- **Не сдерживай себя** — если можешь продвинуться на 3 этапа за день, делай
- **Supabase** вместо Google Sheets / Airtable — масштабируется лучше
- **OpenRouter** — единый API ко всем моделям
