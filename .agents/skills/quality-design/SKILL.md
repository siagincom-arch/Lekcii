# Quality Design Skill

Этот скилл обеспечивает создание качественного, профессионального дизайна для веб-страниц, лендингов и приложений. Основан на проверенных принципах UX/UI, маркетинга и визуальной иерархии.

## Когда использовать

- Создание **лендинга** или главной страницы
- **Редизайн** существующего интерфейса
- Улучшение **визуального качества** сайта
- Разработка **фронтенда** для продукта
- Любая задача, связанная с **UI/UX, дизайном, стилями, вёрсткой**

## Пошаговый алгоритм работы

### Шаг 1: Изучить проект

Перед созданием дизайна **обязательно** изучи весь проект:
- Что за продукт / сервис?
- Для кого он? (целевая аудитория)
- Какие ключевые функции?
- Какие цвета, стили уже заданы в проекте?

> Без понимания продукта дизайн будет шаблонным.

### Шаг 2: Проработать структуру страницы

Используй проверенную структуру лендинга:

```
1. Хедер (навигация, логотип, CTA-кнопка)
2. Hero-секция (главный экран с оффером)
3. Блок доверия (цифры, достижения, партнёры)
4. Преимущества (3–4 ключевых пункта)
5. Основные фичи (описание функционала с визуалом)
6. Демонстрация (скриншоты, видео, интерактив)
7. Тарифы (если применимо)
8. Отзывы (если есть)
9. CTA-блок (призыв к действию)
10. Футер (контакты, ссылки, копирайт)
```

Не начинай кодить, пока структура не согласована.

### Шаг 3: Написать текст по методологии JTBD

**Jobs to be Done** — пиши текст от задач пользователя, а не от функций продукта:

| ❌ Плохо (от функций) | ✅ Хорошо (от задач JTBD) |
|---|---|
| «У нас 10 000 тестов» | «Подготовься к ЕГЭ на 90+ баллов» |
| «Интеграция с 50 сервисами» | «Автоматизируй рутину за 5 минут» |
| «AI-powered engine» | «Получи ответ за секунду вместо часа поиска» |

Принципы JTBD для текстов:
- **Заголовок** = результат, который получит пользователь
- **Подзаголовок** = как именно это работает (коротко)
- **Каждый блок** отвечает на одну конкретную «работу» пользователя

### Шаг 4: Применить иерархию акцентов

> **Главный принцип: Где подсветить — там и внимание.**

Правила визуальной иерархии:
1. **Самое важное** — крупный шрифт, жирный, контрастный цвет
2. **Второстепенное** — средний шрифт, обычная толщина
3. **Детали** — мелкий шрифт, приглушённый цвет
4. **CTA-кнопки** — яркий акцентный цвет, выделяются на фоне
5. **Не более 2–3 акцентов** на один экран (viewport)

Реализация в CSS:
```css
/* Иерархия акцентов */
.hero-title {
  font-size: clamp(2.5rem, 5vw, 4rem);
  font-weight: 800;
  line-height: 1.1;
}

.hero-subtitle {
  font-size: clamp(1.1rem, 2vw, 1.4rem);
  font-weight: 400;
  opacity: 0.8;
}

.cta-button {
  font-size: 1.1rem;
  font-weight: 600;
  padding: 1rem 2.5rem;
  border-radius: 12px;
  /* Акцентный цвет — выделяется */
}
```

### Шаг 5: Соблюдать правила UX и UI

**UI (User Interface)** — визуальная красота:
- Гармоничная **цветовая палитра** (не более 3–4 цветов)
- Современная **типографика** (Google Fonts: Inter, Outfit, Roboto)
- **Микро-анимации**: hover-эффекты, плавные переходы (transition 0.3s)
- **Градиенты** и тени для глубины
- **Glassmorphism** при необходимости (backdrop-filter: blur)
- **Отступы и воздух** — не загромождай (padding: 80px–120px между секциями)

**UX (User Experience)** — логика и удобство:
- Кнопки **на заметных местах** — пользователь не должен искать
- **Один главный CTA** на экран — не путай пользователя
- **Навигация** — понятная, не более 5–7 пунктов
- **Mobile-first** — сначала мобильная версия, потом десктоп
- **Скорость загрузки** — оптимизируй изображения, используй lazy loading
- **Читаемость** — контрастный текст, строка не длиннее 65–75 символов

### Шаг 6: Адаптивный дизайн (Responsive)

Всегда делай адаптацию под мобильные устройства:

```css
/* Базовые брейкпоинты */
@media (max-width: 768px) {
  /* Мобильные стили */
  .hero-title { font-size: 2rem; }
  section { padding: 40px 20px; }
}

@media (max-width: 1024px) {
  /* Планшеты */
}
```

## Чеклист качества дизайна

Перед завершением работы проверь:

- [ ] **Иерархия акцентов** — есть ли чёткая визуальная иерархия?
- [ ] **JTBD** — тексты написаны от задач пользователя?
- [ ] **CTA-кнопки** — видны, контрастны, с понятным текстом?
- [ ] **Мобильная версия** — адаптация работает?
- [ ] **Типографика** — подключён современный шрифт (Google Fonts)?
- [ ] **Цвета** — палитра гармонична, не более 3–4 основных цветов?
- [ ] **Отступы** — достаточно воздуха между секциями?
- [ ] **Анимации** — есть плавные hover-эффекты и переходы?
- [ ] **Скорость** — изображения оптимизированы?
- [ ] **Нет «шаблонности»** — дизайн выглядит уникально?

## Промпт-шаблоны

### Для создания лендинга с нуля:
```
Изучи проект и его функционал. Создай главную страницу:
1. Hero-секция с главным оффером (крупный заголовок + подзаголовок + CTA)
2. Блок преимуществ (3–4 карточки с иконками)
3. Описание основных функций с визуалом
4. Демонстрация продукта (скриншот или видео)
5. Призыв к действию + форма регистрации
6. Футер

Текст по методологии Jobs to be Done.
Соблюдай иерархию акцентов, UX и UI.
Адаптивный дизайн (mobile-first).
Современный стиль: градиенты, тени, микро-анимации.
```

### Для улучшения существующего дизайна:
```
Улучши текущий дизайн страницы:
1. Добавь иерархию акцентов (крупные заголовки, контрастные CTA)
2. Перепиши тексты по методологии JTBD
3. Добавь микро-анимации (hover, transitions)
4. Улучши мобильную адаптацию
5. Добавь воздух между секциями
6. Используй современный шрифт (Google Fonts)
Соблюдай UX и UI.
```

### Для работы с референсами:
```
Вот референс дизайна [скриншот/ссылка].
Сделай похожий лендинг для моего проекта:
- Сохрани стиль и визуальную иерархию
- Адаптируй тексты под мой продукт по JTBD
- Добавь адаптивный дизайн
- Соблюдай UX и UI
```

## Правила, которые нельзя нарушать

1. **Никогда не делай шаблонный дизайн** — каждый проект уникален
2. **Убеждает логика, а не только красота** — UX важнее UI
3. **Оптимизация изображений** — в конце, не в начале
4. **Не перегружай** — один экран = один посыл
5. **Mobile-first** — 70%+ трафика с мобильных устройств
