### Вопрос 2
**Что такое «иерархия акцентов» и почему это главный принцип дизайна?**

<details>
<summary>✅ Ответ</summary>

- **Иерархия акцентов** — принцип, по которому внимание пользователя направляется туда, где элементы выделены (жирный текст, крупный шрифт, контрастный цвет)
- Заменяет все сложные теории (F-паттерн, боли и т.д.)
- **«Где подсветить — там и внимание. Всё.»**
- Достаточно прописать в промпте: «Соблюдай иерархию акцентов»
</details>

---

### Вопрос 3
**В чём разница между UX и UI?**

<details>
<summary>✅ Ответ</summary>

| | UI (User Interface) | UX (User Experience) |
|---|---|---|
| **Суть** | Внешний вид, красота | Удобство использования |
| **Фокус** | Кнопки, цвета, анимации, шрифты | Куда нажимает, что видит, логика пути |
| **Аналогия** | «Красота в хаосе» | «Логика и удобство» |

**Рекомендация:** Всегда прописывать в промптах: «Соблюдай UX и UI»
</details>

---

### Вопрос 4
**Что такое Jobs to be Done (JTBD) и зачем его использовать для текстов на сайте?**

<details>
<summary>✅ Ответ</summary>

- **JTBD** — методология, ориентированная на решение проблем пользователей
- Тексты пишутся не «про продукт», а «про задачу пользователя», которую продукт решает
- Помогает создать **убедительный** лендинг, потому что пользователь сразу видит свою проблему и решение
- Есть расширенная версия — **Advanced JTBD**
- Промпт: «Напиши тексты по методологии Jobs to be Done»
</details>

---

### Вопрос 5
**Почему Google AI Studio рекомендуется для дизайна, а не Antigravity?**

<details>
<summary>✅ Ответ</summary>

- Google AI Studio **специализируется на фронтенде** — делает его качественнее
- Поддерживает фреймворки: React, Angular, **Next.js**
- Результат легко **перенести в Antigravity** (скачать код или через GitHub)
- Antigravity по-прежнему лучше для **кодинга и бэкенда**
- Комбинация: Google AI Studio для дизайна → Antigravity для интеграции
</details>

## Блок 2: Ветки GitHub

### Вопрос 6
**Зачем нужны ветки (branches) в GitHub?**

<details>
<summary>✅ Ответ</summary>

- Для **безопасной командной работы** — каждый работает в своей ветке
- Основная ветка **Main** не затрагивается при разработке
- После проверки изменения **мерджатся** (объединяются) в Main
- Это защищает от ситуаций, когда кто-то **сломает** или **удалит** что-то в основном коде
- Можно использовать для **A/B-тестирования** и **собственной безопасности**
</details>

---

### Вопрос 7
**Опишите порядок действий: как создать ветку, внести изменения и отправить их.**

<details>
<summary>✅ Ответ</summary>

1. Скачать (развернуть) репозиторий: «Разверни репозиторий GitHub по ссылке»
2. Создать ветку: «Создай отдельную ветку, назови её *сайт*»
3. Внести изменения (например, обновить дизайн главной страницы)
4. Отправить в ветку: «Отправь код в ветку *сайт*»
5. Главный разработчик проверяет ветку на GitHub
6. Если всё ок — нажимает **Merge** для объединения с Main
</details>

### Вопрос 9
**Как перенести дизайн из Google AI Studio в Antigravity?**

<details>
<summary>✅ Ответ</summary>

**Способ 1: Через скачивание**
1. Google AI Studio → Code → Скачать
2. Разархивировать → закинуть папку в Antigravity

## Блок 4: Бизнес-цикл

### Вопрос 10
**Какая существует статистика по успешности бизнес-гипотез?**

<details>
<summary>✅ Ответ</summary>

- **1 из 10** гипотез выстреливает (стандартная статистика)
- **3 из 100** — на большой выборке
- Стоимость одной гипотезы раньше: **~1 млн ₽**
- Сейчас (благодаря вайбкодингу): **~300-500 тыс. ₽**
- **Отказ от нерабочей идеи** — тоже экономия!
</details>

### Вопрос 11
**Назовите три типа исследований для валидации идеи продукта.**

<details>
<summary>✅ Ответ</summary>

1. **Качественные** — личное общение с аудиторией (нельзя делегировать, основатель должен общаться лично)
2. **Количественные** — рассылка анкет с 3–4 вопросами, сбор обратной связи
3. **Онлайн-разведка (OSINT)** — добыча информации из открытых источников. Есть специальные инструменты на GitHub
</details>

   - в) Формат файлов для дизайна
