## 🟢 Блок 1 — Базовые понятия (тест)

### Вопрос 1.1
**Что такое репозиторий (repository)?**

- A) Программа для написания кода
- B) Хранилище файлов проекта с историей изменений
- C) Сервер для запуска приложений
- D) Система общения между программистами

<details>
<summary>✅ Ответ</summary>

**B) Хранилище файлов проекта с историей изменений.**

Репозиторий — это место, где хранится весь код проекта, включая полную историю всех изменений (коммитов).
</details>

---

### Вопрос 1.2
**Какая команда показывает текущее состояние файлов в Git?**

- A) `git show`
- B) `git log`
- C) `git status`
- D) `git check`

<details>
<summary>✅ Ответ</summary>

**C) `git status`**

Эта команда показывает, какие файлы изменены, добавлены или ещё не отслеживаются.
</details>

---

### Вопрос 1.3
**Для чего используется команда `git push`?**

- A) Скачать код с GitHub на компьютер
- B) Добавить файлы в отслеживание Git
- C) Отправить локальные коммиты на удалённый сервер (GitHub)
- D) Создать новую ветку

<details>
<summary>✅ Ответ</summary>

**C) Отправить локальные коммиты на удалённый сервер (GitHub).**

`git push` — это загрузка ваших изменений из локального репозитория на GitHub.
</details>

---

### Вопрос 1.4
**Что произойдёт, если выполнить `git push --force`?**

- A) Git откажет в отправке
- B) Удалятся все файлы на GitHub
- C) Версия на GitHub будет принудительно заменена вашей локальной версией
- D) Создастся новая ветка

<details>
<summary>✅ Ответ</summary>

**C) Версия на GitHub будет принудительно заменена вашей локальной версией.**

⚠️ Флаг `--force` затирает всё, что было на сервере. Безопасно только если вы уверены, что на сервере ничего ценного нет.
</details>

## 🟡 Блок 2 — Порядок команд (восстанови последовательность)

### Вопрос 2.1
**Расставь команды в правильном порядке для загрузки кода на GitHub:**

Команды (перемешаны):
- `git push -u origin main`
- `git add .`
- `git clone https://github.com/user/repo.git`
- `git commit -m "Initial commit"`
- `git status`

<details>
<summary>✅ Правильный порядок</summary>

```
1. git clone https://github.com/user/repo.git
2. (скопировать файлы проекта в папку репо)
3. git add .
4. git status
5. git commit -m "Initial commit"
6. git push -u origin main
```
</details>

---

### Вопрос 2.2
**Что нужно сделать ПЕРЕД первым `git commit`, если Git его не принимает?**

<details>
<summary>✅ Ответ</summary>

Настроить имя и email:

```powershell
git config --global user.name "Ваше Имя"
git config --global user.email "your-email@example.com"
```

Git требует идентификацию автора для каждого коммита.
</details>

## 🧠 Блок 4 — Найди ошибку

### Вопрос 4.1
Студент пытается залить код на GitHub и получает ошибку. Найди проблему:

```powershell
git add .
git push -u origin main
```

<details>
<summary>✅ Ответ</summary>

**Пропущен коммит!** Между `git add` и `git push` обязательно нужен `git commit`:

```powershell
git add .
git commit -m "описание изменений"    # ← пропущено!
git push -u origin main
```

Без коммита файлы добавлены в staging area, но не зафиксированы.
</details>

---

### Вопрос 4.2
Студент видит такую ошибку:

```
Author identity unknown
*** Please tell me who you are.
```

Что нужно сделать?

<details>
<summary>✅ Ответ</summary>

Настроить Git-идентификацию:

```powershell
git config --global user.name "Имя"
git config --global user.email "email@example.com"
```
</details>

---

### Вопрос 4.3
Студент сделал `git push`, но получил:

```
error: src refspec main does not match any
```

В чём причина?

<details>
<summary>✅ Ответ</summary>

**Нет ни одного коммита!** Ветка `main` ещё не существует, потому что не было сделано ни одного коммита. Нужно сначала:

```powershell
git add .
git commit -m "Initial commit"
git push -u origin main
```
</details>
