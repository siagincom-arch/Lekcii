## 1. Работа с GitHub-репозиториями

### 1.1 Создание репозитория
- Заходим на **GitHub** → создаём новый репозиторий (public или private)
- Получаем **ссылку** вида `https://github.com/username/repo-name.git`
- Эта ссылка — ключевой инструмент для всех операций

### 1.2 Первоначальная загрузка кода (push)
Пошаговый процесс заливки проекта на GitHub:

```powershell
# 1. Клонируем пустой репозиторий
git clone https://github.com/username/repo.git

# 2. Копируем файлы проекта в папку репозитория
Copy-Item -Path "путь/к/проекту/*" -Destination "путь/к/репо/" -Recurse

# 3. Переходим в папку репозитория
cd путь/к/репо

# 4. Добавляем все файлы в отслеживание
git add .

# 5. Проверяем статус (зелёные файлы = готовы к коммиту)
git status

# 6. Фиксируем изменения (коммит)
git commit -m "Описание изменений"

# 7. Отправляем на GitHub
git push -u origin main
```

### 1.3 Настройка Git-идентификации
Перед первым коммитом Git требует указать **имя** и **email**:

```powershell
git config --global user.name "Ваше Имя"
git config --global user.email "ваш-email@example.com"
```

### 1.4 Принудительный push (force)
Если на GitHub уже есть изменения, а у вас свежий проект:

```powershell
git push -u origin main --force
```

> ⚠️ `--force` затирает всё на сервере. Использовать осторожно!

## 2. Скачивание и развёртывание чужих репозиториев

### 2.1 Клонирование
- Берём ссылку с GitHub (зелёная кнопка **Code**)
- Отправляем в Antigravity: `разверни код в моей директории`
- Antigravity выполняет `git clone` и разворачивает проект

### 2.2 Поиск готовых проектов
- Использовать **Perplexity** с интеграцией GitHub для поиска open-source проектов
- Можно искать прямо по описанию: *«поиск в репозиториях GitHub, открытый код для [тема]»*
- Найденные проекты можно клонировать и адаптировать

> 💡 **Важно:** Если проект выставлен как open-source, автор хотел, чтобы им пользовались. Можно брать и адаптировать.

### 2.3 Запуск чужого проекта
- После клонирования может потребоваться установка **токенов** (API-ключи)
- Пишем Antigravity: `запусти проект`
- Если нужны токены — бот запросит их

## 3. Управление репозиториями

### 3.1 Обновление репозитория
- Вносим изменения → `git add .` → `git commit -m "описание"` → `git push`
- Antigravity: `обнови репозиторий`

### 3.2 Удаление кода из репозитория
- Можно попросить Antigravity удалить файлы и сделать коммит
- Для удаления **самого репозитория** — через настройки GitHub вручную

### 3.3 Приватность репозитория
- **Settings** → в самом низу **Change repository visibility**
- Можно переключать между **public** и **private**

### 3.4 Токен GitHub
- Задаётся один раз **на уровне аккаунта**
- После этого не нужно давать токен для каждого нового репозитория
- Antigravity хранит токены в зашифрованном контейнере (`.gitignore`)

## 4. Организация рабочей среды

### 4.1 Принцип: один репозиторий = один проект
- Не смешивать разные проекты в одном репо
- Для каждого нового проекта — отдельный репозиторий

### 4.2 Структуризация папок
- Можно попросить Antigravity проанализировать структуру и дать рекомендации
- Создание и перемещение папок частично можно автоматизировать
- Переименование кириллических папок иногда вызывает проблемы

### 4.3 Методология пошагового подхода
- Создаём папку → создаём `.md` файлы (идея, ТЗ, архитектура...)
- Буксируем (drag & drop) файлы в чат Antigravity для контекста
- Двигаемся строго по шагам, не перескакиваем

> 💡 **Совет от участника:** *«Если я чего-то не умею, я делаю так, как мне говорят. Не перепрыгиваю шаги. Степ-бай-степ.»*

## 5. Решение проблем Windows

### 5.1 Конфликт терминала
- На Windows Antigravity иногда зависает из-за неправильных команд терминала
- **Решение:** установить скилл для фикса PowerShell-команд

```powershell
npx skills find windows terminal fix
```

### 5.2 Что делает этот скилл
- Обучает агента понимать, что он работает в **Windows**
- Заставляет использовать правильные команды **PowerShell**
- Альтернативный подскилл: добавляет поддержку Linux-утилит в Windows

### 5.3 Глобальные скиллы
- ⚠️ Устанавливать осторожно! Чрезмерная установка может вызвать зависание
- Рекомендация: сначала спросить Antigravity, какие скиллы нужны для проекта
- Устанавливать по одному и проверять
